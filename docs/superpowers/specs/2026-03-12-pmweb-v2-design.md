# PMWEB v2 — 多层级任务、拖拽、里程碑优化

## 概述

在 PMWEB v1 基础上进行重大改造：取消项目/任务区分，改为最多 3 层的统一任务树；甘特图支持拖拽调整时间和布局；里程碑仅在图表中展示和操作；增加任务列表与图表的联动高亮；固化数据格式支持版本迁移。

## 需求清单

1. 取消项目和任务区分，支持最多 3 层的统一任务列表
2. 甘特图时间线支持手动拖拽（平移、拉伸边缘、里程碑移动、同行合并）
3. 里程碑仅在甘特图和日历图中展示，不在任务列表中体现，可直接在图表中增删改
4. 里程碑图标颜色优化，与任务条明确区分
5. 固化导入导出文件格式，支持版本号 + 自动迁移
6. 任务列表与图表联动高亮

## 数据模型 v2

```json
{
  "pmweb_version": 2,
  "exported_at": "2026-03-12T14:30:00",
  "tasks": [
    {
      "id": "uuid",
      "parentId": null,
      "name": "任务名称",
      "description": "",
      "assignee": "",
      "color": "#4a90d9",
      "startDate": "2026-03-01",
      "endDate": "2026-03-15",
      "status": "todo",
      "order": 0,
      "ganttRow": null
    }
  ],
  "milestones": [
    {
      "id": "uuid",
      "taskId": "uuid",
      "name": "里程碑名称",
      "date": "2026-03-05",
      "icon": "◆"
    }
  ],
  "settings": {
    "lastView": "gantt",
    "lastGranularity": "month"
  }
}
```

### 字段说明

**Task:**
- `parentId`: null 表示顶层任务，否则为父任务 ID。最多 3 层嵌套。
- `color`: 每个任务可独立设置颜色。子任务默认继承父任务颜色，可单独覆盖。
- `startDate`/`endDate`: 叶子任务必填。父任务可为 null，此时自动取子任务的 min(startDate)~max(endDate)。
- `ganttRow`: 甘特图行号。null 表示独占一行。相同值的同级任务合并到同一行显示。
- `status`: "todo" | "inProgress" | "done"

**Milestone:**
- `taskId`: 关联的任务 ID（必填）。
- `icon`: "◆" | "★" | "●" | "▲"

### 树形结构约束

- 最多 3 层：顶层(depth=0) → 二层(depth=1) → 三层(depth=2)
- 添加子任务时检查父任务深度，depth >= 2 时禁止添加
- 删除父任务时级联删除所有子任务和关联里程碑

## 界面改造

### 工具栏

```
┌─────────────────────────────────────────────────────┐
│  [导入] [导出]  |  [甘特图/日历]  |  [年|月|周]       │
```

- 删除顶部"+ 新建任务"按钮

### 左侧任务列表

```
任务列表
───────────────────────
▼ 需求跟踪          [+][⋯]
  ▼ 用户模块        [+][⋯]
      登录功能       [⋯]

```

- 每个任务节点 hover 时显示操作按钮：
  - `[+]`: 添加子任务（仅 depth 0-1 的任务显示此按钮）

### 任务列表 ↔ 图表联动高亮

- **hover 联动**: 悬停任务列表项 → 甘特图/日历中对应任务条高亮（加亮 + 边框发光）
- **点击联动**: 点击选中任务 → 对应任务条持续高亮，甘特图自动滚动到可见位置
- **反向联动**: 点击甘特图中的任务条 → 左侧列表对应项高亮

## 甘特图拖拽

### 整体平移

- 鼠标按住任务条中间区域，左右拖动
- 拖动时显示半透明预览和日期 tooltip（显示新的起止日期）
- 释放后更新 startDate/endDate，保持时长不变
- 父任务时间范围自动重新计算

### 边缘拉伸

- 鼠标靠近任务条左右边缘 5px 内，光标变为 `col-resize`
- 拖拽左边缘调整 startDate，拖拽右边缘调整 endDate
- 拉伸时 tooltip 显示变更中的日期
- 不允许 endDate < startDate（最短 1 天）

### 里程碑拖拽

- 按住里程碑图标左右拖动，调整日期
- 任务级里程碑约束在所属任务的时间范围内
- tooltip 显示目标日期

### 同行合并拖拽

- 上下拖拽任务条到另一个同级任务的行上
- 拖拽时目标行高亮提示
- 释放后两个任务共享同一 `ganttRow` 值
- 时间重叠的任务不允许合并到同一行（释放时弹回）
- 再次上下拖离可恢复独占行

### 通用规则

- 拖拽中按 Esc 取消
- 所有拖拽操作立即保存到 localStorage

## 里程碑交互（仅图表中操作）

### 甘特图中

- 双击任务条空白位置 → 弹出"添加里程碑"对话框
- 点击已有里程碑 → 弹出编辑框
- 右键里程碑 → 删除选项

### 日历中

- 点击已有里程碑 → 编辑
- 日期格子右键菜单增加"添加里程碑"选项

## 里程碑视觉优化

- 里程碑图标统一使用固定红色 `#E74C3C`，不跟随任务颜色
- 加 2px 白色描边（`text-stroke` 或 `filter: drop-shadow`），确保在任何颜色背景上醒目
- hover 时放大 1.3 倍 + 显示 tooltip（名称 + 日期）
- 日历视图中同样用红色图标

## 数据格式固化与迁移

### 导出文件格式

```json
{
  "pmweb_version": 2,
  "exported_at": "ISO-8601 时间戳",
  "tasks": [...],
  "milestones": [...],
  "settings": {...}
}
```

### 迁移策略

- `pmweb_version` 为必填整数字段
- 导入时读取版本号，对照内置迁移函数链依次升级
- 无法识别的版本号（过高）→ 提示"数据版本过新，请升级软件"
- localStorage 与导出文件格式一致，启动时检查版本号并自动迁移

### v1 → v2 迁移

```
输入: { version:1, projects:[], tasks:[], milestones:[], settings:{} }
输出: { pmweb_version:2, tasks:[], milestones:[], settings:{} }

步骤:
1. projects[] 中每个 project → 新 task（parentId:null, color:project.color）
2. tasks[] 中每个 task → 新 task（parentId: 对应 project 转化后的 taskId）
3. milestones[] 中 projectId → 转为对应顶层 task 的 taskId
4. 新增字段补默认值：ganttRow:null
5. version 字段改为 pmweb_version:2
```

## 技术约束

- 保持单文件 HTML 应用，零依赖
- 不使用 ES6 Module
- 拖拽使用原生 mousedown/mousemove/mouseup 事件
- 信创浏览器兼容（flexbox + 绝对定位，避免新 CSS 特性）
